# Project Context
Ruby 3.4.9, Rails 8, PostgreSQL
API-only: false (full-stack with Hotwire)
Authentication: generated by rails authentication

# Conventions
- Use strong parameters in controllers, never permit all
- Fat models, skinny controllers
- Service objects in app/services/ for complex business logic
- Form objects in app/forms/ for multi-model forms
- Query objects in app/queries/ for complex scopes
- Service objects live in app/services/
- Presenters live in app/presenters/
- Use Interactor gem for multi-step business logic
- All controllers should be thin; push logic to service objects
- Prefer named scopes over raw where
